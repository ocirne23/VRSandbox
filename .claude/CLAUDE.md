# Project Description
A modern c++ Game Engine codebase. Windows/MSVC only. The `App` executable is the testbed: it spawns `Entities/SponzaScene.pre` (Sponza + skysphere prefabs, Sponza with a static mesh collider) and `Entities/character.pre` (skinned, animated, scripted character) through the `World`, runs an ImGui editor UI (scene hierarchy, properties, asset browser, visual-script node editor, tweaks) around the viewport, and drives a transform gizmo. VR is initialized with `EVr::DISABLED` in main.cpp â€” flip to `ENABLED` to run on an OpenXR headset. A Bistro scene (`Entities/BistroScene.pre`) exists as an alternative test scene.

## Building
* CMake with Visual Studio 18 2026 generator, build dir is `Build/`
	* Reconfigure: `cmake -B Build` (only needed when files are added/removed â€” all sources are globbed, never edit CMakeLists to add files)
	* Build: `cmake --build Build --config Debug` (also `RelWithDebInfo`, `Release`)
	* Output: `Build/x64/<Config>/App.exe`. Run from anywhere inside the repo; `FileSystem::initialize()` walks up to find `Assets/` and sets it as the working directory, so all asset paths are relative to `Assets/`
* No test suite, no linter. Verifying = it compiles + the user runs it
* Dependencies are prebuilt in `Dependencies/` (Include/Lib/Dll) â€” Vulkan-Hpp (no exceptions, no constructors), SDL3, ImGui docking branch, Assimp, glslang/shaderc, OpenXR loader, Nsight Aftermath, box3d, Steam Audio (static, no DLL; phonon(d).lib bundles its pffft/mysofa/zlib deps), miniaudio (single header, no build), meshoptimizer (LOD generation). box3d, steam-audio and meshoptimizer source stay vendored under `Dependencies/` for rebuilding the prebuilt libs â€” recipes in `Dependencies/CMakeLists.txt` (which is otherwise unused; everything links the prebuilt `<name>.lib`/`<name>d.lib` pairs)

## Style
* Using exclusively c++20 modules instead of headers. (.ixx instead of .h)
	* Most libraries use module partitions: `export module RendererVK:Renderer;` in the interface .ixx, `module RendererVK;` in the implementation .cpp; public surface re-exported from `Public/<Lib>.ixx` (`RendererVK`, `Entity`, `Animation`, `File`, `Input` all do this)
	* `Core` and `UI` use dotted module names instead (`Core.Tweaks`, `UI.NodeEditor.Scene`, one module per .ixx)
	* `Public/` folder .ixx's are importable from outside the library; `Private/` is internal
	* The `Core` module re-exports std headers as header units â€” if you need a std header that isn't available, add it to `Code/Core/Public/Core.ixx`
	* `Core.fwd` / `*.fwd.ixx` modules hold forward declarations
	* One exception: `Script/Public/ScriptAPI.h` is a plain header â€” it's the ABI shared with runtime-compiled script DLLs and is `#include`d on both sides
* Modernest conventions
	* No new/delete, only memory safe types.
	* Raw pointers are non-owning
	* Minimal comments, only in exceptional cases where code intent is too difficult to communicate
	* Prioritize performance
* Engine singletons live in `export namespace Globals` inside the owning .ixx (`Globals::rendererVK`, `Globals::device`, `Globals::input`, `Globals::ui`, `Globals::world`, `Globals::physics`, `Globals::scriptHost`, `Globals::scriptEvents`, `Globals::assetRegistry`, `Globals::time`, `Globals::allocator`, ...). RendererVK globals use `#pragma init_seg` to control construction order â€” keep that when adding one. Globals in *different* libraries have no defined construction order relative to each other (see `ScriptEventManager::initialize` for the pattern when one global must register with another)
* int8/uint64-style typedefs from `Core`; glm via `Core.glm`; `assert` is compiled out in non-debug via forceinclude.h

## RendererVK
* Modern Vulkan renderer designed to only use modern features (minimum spec RTX 2000 series)
* Device Generated Commands (DGC) for everything
* Lights culled with a world space hash table indexing into grids with a cellcount based on distance to camera
* Global illumination using scrolling clipmap radiance cascades (ray traced probes)
* RTAO, TAA, PCSS cascades, volumetric fog, GPU compute skinning
* Everything upstream of composite is linear HDR: `EyeAdaptationPipeline` (log-luminance histogram + smoothed auto-exposure, compute) feeds `CompositePipeline` (fullscreen HDRâ†’display pass into the swapchain before ImGui; tonemap operators: off/Reinhard/ACES/AgX)
* Frame order (see `recordCommandBuffers()` / `present()`): skinning compute â†’ indirect cull â†’ light grid â†’ particle sim compute â†’ shadow cull + draw â†’ static mesh/G-buffer â†’ GI probes â†’ RTAO â†’ volumetric fog â†’ TAA â†’ eye adaptation â†’ composite + ImGui. Inside the scene-color pass: static mesh forward â†’ decals â†’ GI-probe debug â†’ debug lines â†’ particles â†’ fog apply
* Layer structure under `Private/`:
	* `Objects/` â€” thin Vulkan object wrappers (Device, SwapChain, Buffer, Pipeline, AccelerationStructure, GBuffer, SceneColor, ShadowMap, ...)
	* `Pipeline/` â€” one class per render pass/feature (GBuffer, GIProbe, RTAO, TAA, VolumetricFog, EyeAdaptation, Composite, Skinning, light grid + indirect/shadow culling compute, shadows). Each registers its own Tweak vars
	* `Data/` â€” MeshDataManager, TextureManager, StagingManager, ShaderDatabase (Aftermath)
	* `Renderer.ixx/.cpp` â€” orchestrates everything; per-pass `record*()` methods called from `recordCommandBuffers()`
	* `OpenXRSession.ixx` â€” VR: OpenXR session/swapchain wrapper (`Globals::openXR`, implements `IVrSession`). Two-phase init around Vulkan device creation (runtime dictates instance/device extensions + physical device); stereo rendering targets a 2-layer multiview swapchain; in VR the editor viewport sub-rect is ignored (full-extent render). `Renderer::initialize` takes `EVr::ENABLED/DISABLED`
* Scene objects: an `ObjectContainer` wraps one loaded `ISceneData` (meshes/materials/textures, optional `MaterialOverrides`); `container.spawnNodeForIdx()` returns a `RenderNode` instance handle (RAII, move-only), `spawnSkinnedNode()` the GPU-skinned variant (an `AnimatorComponent` feeds its bone palette via `allocateSkinningPalette`/`setSkinningPalette`). Destroying a `RenderNode` recycles its resources without GPU sync (all per-frame data is re-uploaded anyway): transform slots go on a free list, a skinned node's whole allocation bundle (`Renderer::SkinnedInstanceBundle`) is parked in place and reused by the next `spawnSkinnedNode` of the same container, and rebased sub-node instance offsets are uploaded once per NodeSpawnIdx and shared. Container-level resources still live for the app's lifetime (`~ObjectContainer` TODO). Lights are added per-frame via `renderer.addPointLight/addSpotLight/addAreaLight/addLightInfo/...`
* Per-frame instance flow: `renderNode(node, passMask)` is LOCK-FREE (callable from any job between beginFrame/present; atomic cursor claims, monotonic — on overflow the tail drops for ONE frame, `m_instanceOverflowStart` CAS-min records the first failed claim and present() clamps the counter to it, capacity regrows next beginFrame; NEVER roll a bump cursor back). `addLightInfo/addFogVolume` are atomic-cursor lock-free, `addDebugLine` + the transform dirty lists are `PerWorker`-staged and merged in present(); Renderer::initialize requires the JobSystem first. It writes the node's instances + a per-node pass mask (`PASS_MAIN/SHADOW/GI`, Layout.ixx = Spatial's pass bits) into mapped per-frame buffers; the main indirect cull, shadow cull, and GI TLAS-instance writer each early-out on their bit (TLAS also range-bounds by `RT/GI / TLAS Range`). Render node transforms upload SPARSELY: write only via `RenderNode::setTransform` (change-detected, per-frame-in-flight dirty lists drained in `present()`) - a mutable getTransform would bypass the tracking
* Texture mip streaming (`TextureStreamer`, "Streaming" tweaks): DDS textures load tail-only (mips <= 128px) and stream mips in/out of a fixed VRAM budget by camera priority (projected-size pass in `renderNode`). Residency changes recreate the image and rewrite that slot in all 5 bindless texture arrays (UPDATE_AFTER_BIND, applied per frame slot in `recordCommandBuffers` after the fence wait); disk reads run on a worker thread; old images retire after NUM_FRAMES_IN_FLIGHT. Non-DDS/procedural/embedded textures are pinned at full res
* Mesh data streaming (`MeshStreamer`, "Streaming" tweaks): cooked scenes register each source mesh + its generated LOD levels as a "mesh set" carrying the byte ranges its data occupies in the .vsc cache; the vertex/index mega-buffers are free-listed (`MeshDataManager`, BitRangeAllocator in element-aligned buckets). When resident bytes exceed "Mesh budget (MB)", the least-recently-referenced sets unseen for "Mesh cold frames" evict: their MeshInfos get indexCount 0 (DGC draws/shadow/TLAS writes become no-ops with no re-record; static BLASes persist - they are snapshots of identical geometry) and their ranges return to the free list after NUM_FRAMES_IN_FLIGHT. An evicted set referenced again re-streams from the .vsc on a worker thread (attributes re-interleaved to MeshVertex) and restores its MeshInfos at fresh offsets; the LOD selector renders the nearest resident level meanwhile. BLASes evict and rebuild with their set, and a LOD chain shares ONE BLAS at its "RT/BLAS LOD level" (alias table in AccelerationStructure; the TLAS-instance writer packs the aliased RT meshIdx into each instance's sbtOffset, which hit shaders use for geometry fetches while materials stay per-instance); static BLASes are copy-compacted a few frames after build ("RT/BLAS compaction", fence-timed size queries, ~30-50% back) Only cooked static meshes stream - skinned/procedural/direct-import data is pinned. NOTE: with Spatial culling Off everything is referenced every frame, so only despawned containers and Col_ proxies ever go cold
* Mesh LODs ("LOD" tweaks): meshes/nodes named `LodN_<name>` form authored chains grouped under `<name>` (a `Lod0_X` node path also resolves as `X`; level>0 never instanced, like `Col_`); static meshes without authored LODs get meshopt-generated index-only chains (shared vertices, "Generated reduction" of the indices per level; pre-baked by the scene cache when the source was cooked, generated at load otherwise). Selection runs per instance ON THE GPU (mesh_lod.inc.glsl in the main + shadow cull shaders; params via UBO lodParams, so tweaks are live): instances are pushed referencing LOD0 and the cull redirects the draw to the selected level's MeshInfo — the CPU sizes every chain member's instance bucket to the chain's full count so any split fits, hysteresis state lives in a per-instance GPU slot range (RenderNode::m_lodStateBase + per-frame node bias buffer; races/garbage benign — clamped into the frame's valid band), the shadow cull selects stateless at 4x/+2 coarser, GI needs no selection (all levels alias one chain BLAS), and per-level pick counts read back to Stats a few frames late. Selection is screen-space error: generated chains carry per-level meshopt simplify errors (mesh-local units, scaled by instance scale, projected to pixels) and use the coarsest level under the "Max error (px)" budget — object size needs no separate tuning; authored chains without error data fall back to the projected-diameter/"Full-res pixels" metric. renderNode keeps every level of a referenced chain warm in the mesh streamer (once-per-group frame stamp), and the cull falls back to the nearest resident level (indexCount 0 = evicted) while a re-stream lands. Skinned meshes get bind-pose-generated chains too (index-only over the per-instance deformed output regions, which preserve source vertex ordering; per-instance MeshLodGroups registered at spawnSkinnedNode, recycled with the bundle; the per-frame skinned BLAS builds at the chain's "RT/BLAS LOD level"). Skinned output MeshInfos are excluded from static BLAS builds/compaction (uninitialized until the skinning compute runs; their addresses are owned per frame slot by the skinned rebuild)
* Try to record CommandBuffers once (not per frame) whenever possible; call `setHaveToRecordCommandBuffers()` after changes that invalidate them
* Shaders are GLSL in `Assets/Shaders/`, compiled at runtime with glslang (so shader edits don't need a rebuild â€” F5 in the app calls `reloadShaders()`). `*.inc.glsl` files are includes; `shared.inc.glsl`/`ubo.inc.glsl` define structs that must stay in sync with `Private/Layout.ixx` (`RendererVKLayout`). Compiled SPIR-V + source dumps land in `Assets/Local/` for Aftermath GPU crash analysis

## Entity
* The ECS. Inline component layout: an `Entity` (pos/scale/rot, parent, displayName (`unique_ptr<char[]>`, null when unnamed — `getName()`/`setName()`), spawn template, refcount, `typeBits`, `flags`) and its components live in ONE allocation â€” `typeBits` is a bitmask over the 9 component types (fixed order: Scene, Zone, Cull, Render, Animator, Physics, Audio, Particle, Script — Script stays last; `EComponentID` in `EntityDef.ixx`), and `getComponent<T>(entity)` / `hasComponent<T>` compute the byte offset from it at compile-time-known sizes (`Component.ixx`). Adding a component type means: new struct with `getId()`, entry in `EComponentID` + `inlineSizes` + `componentTypeName` + create/destroy/serialize switches in Entity.cpp + a parse branch in `World::buildTemplate` + a write branch in Prefab.cpp, bump `MaxInlineComponentTypes`
* `EntityPtr`: intrusive refcounted handle (atomic on `refCount`); `Entity::create(template, transform)` placement-news into `Globals::entityAllocator` (thread-safe/lock-free: atomic chunk bump + tag-counted per-size-class Treiber free lists, spinlock only on chunk add; blocks <= `MAX_BLOCK_SIZE` 64KB). Entities form a tree via `SceneComponent::children` + `Entity::parent`; `reparentEntity` moves ownership
* Tree spawns are ONE allocator block: `Entity::create` allocates the template's cached recursive size (`EntitySpawnTemplate::treeAllocSize`, lazy) and the spawn recursion carves per-entity slices from a bump cursor. Freeing: while intact (`EEntityFlag_RootAllocation`/`ContiguousAllocation`) members skip their own free and the root frees the whole block in one call, verified by a refcount pre-walk at root destroy (`contiguousTreeSolelyOwned`); breaks SPLIT the allocation instead of reverting it: reparenting/deleting a member out of its birth tree (`breakContiguousAllocation`) demotes only the path ancestors to per-entity freeing and promotes every off-path subtree (and the departing member) to its own RootAllocation over its contiguous DFS range (subtree sizes come from each child template's cached `treeAllocSize`); the pre-walk failure fallback splits the same way, so an external ref only degrades the path to it recursively. Moving within the same allocation or moving/grafting a whole allocation root never breaks anything
* `Entity::update()` -> `updateTree`: per entity runs ScriptComponent -> AnimatorComponent -> PhysicsComponent, composes the world transform, refreshes the RenderComponent's SpatialIndex entry and pushes its `RenderNode` to the renderer (gated on `SpatialIndex::isVisible` when Spatial/Culling Mode >= SafeCull), recurses into scene children
* Entity state lives in `flags` (`EEntityFlags`), not in components: `EEntityFlag_Enabled` (`isEnabled`/`setEnabled`; off prunes the whole subtree from updateTree and suspends its physics bodies — serialized as an entity-level `Enabled false` next to Position/Rotation/Scale, default true so only the disabled state is authored) and `EEntityFlag_Frozen` (`isFrozen`/`setFrozen`/`isFrozenInTree`; script Update + animator + physics don't tick, script events don't fire — what the Entity Editor sets on an open document, inherited down the tree). Script OnSpawn runs even when frozen. The per-component `enabled` bools (Script/Animator/Physics) are independent of both
* PARALLEL entity update (`World::update(renderer, dt)`, "Entity/Update" tweaks, default on; off = old inline updateTree per root): three phases per frame. (1) Serial main-thread SCRIPT PREPASS (script DLLs/thunks mutate anything freely) that skips subtrees whose cached `SceneComponent::subtreeScripts` is exactly 0 (`SubtreeCountUnknown` = visit; `reparentEntity` resets the new ancestor chain to unknown so moved-in scripts are never skipped). (2) `updateTreeParallel` - the AoS-preserving parallel pass: `parallelFor` over roots (JobCost auto-grained) and, inside the recursion, child subtrees whose last-frame `subtreeEntities` >= "Fork grain" fork as jobs (last forkable child always runs inline; forks claim a per-pass "Max forks in flight" budget so parked parent fibers can never exhaust `numFibers`; parents fiber-wait on their forks; counts remeasured every pass, saturating with unknown). Thread-safe inline: animator tick + `setSkinningPalette` (disjoint palette regions), dynamic-body pose reads, `RenderNode::setTransform`/`renderNode`, `SpatialIndex::updateEntry`, audio follow (per-source atomics). (3) Serial sink flush (`EntityUpdateSinks`, PerWorker-staged): box3d writes (kinematic/static follow + occluder re-bakes, first-update syncs, suspend/resume - flushed by re-running `PhysicsComponent::update` with null sinks) and animator clip-event notifies (arbitrary gameplay hooks). Scripts see pre-pass transforms (cross-entity reads are one frame staler than the old interleaved order); the gizmo entity still uses the serial `Entity::update`
* `World` (`Globals::world`): `spawn("prefabName", transform)`, `spawnAssetFile("Entities/x.pre", ...)`, `createEmptyEntity`. Builds and caches `EntitySpawnTemplate`s (archetype + per-component `SpawnInfo` recipes) from parsed assets; caches `ObjectContainer`s, collision sources/meshes, and retargeted animation clip sets so each source file is imported once. `reloadPrefabs`/`invalidatePrefab` for editing (retired templates stay alive for live entities). OWNS the root entity list (`addRootEntity`/`rootEntities` (const-only)/`clearRootEntities` - all mutation through the World API (InputControls takes a `World&`); main must call clearRootEntities before returning: World is a global, cross-library static teardown order is undefined) and applies all queued `EntityChange`s via `handleEntityChange(change, camera, viewportRect)` (main just pipes UI/script events in; UI notifications route back through `setOnPrefabOpened`/`setOnEntityRespawned` callbacks since the dependency points UI -> Entity)
* `AssetRegistry` (`Globals::assetRegistry`): scans `Assets/` at startup and registers everything by name â€” `ObjectContainer` + `StaticMesh`/`SkinnedMesh` spawnables (.oc), `Animation` clips (.anm), `Animator` graphs (.apl), `Prefab`s (.pre)
* Prefabs: `savePrefab(root, path)` serializes an entity tree back to .pre (per-component `serialize`/`writeRenderSpawnInfo`/...); spawned prefabs are locked instances (`EEntityFlag_PrefabInstance`, unpackable in the Scene panel); `prefabWouldCycle` guards recursive saves
* Components: `SceneComponent` (children), `RenderComponent` (RenderNode + local transform, static or skinned), `AnimatorComponent` (AnimationPlayer + AnimStateMachine from a .apl, ticks and pushes the skinning palette; gameplay via `stateMachine.setFloat/Bool/Trigger`, clip events via `onEvent`), `ScriptComponent` (runs a .scr each frame with the entity as `self`), `PhysicsComponent` (see Physics), `AudioComponent` (named triggerable sounds: `Component Audio` with `Sound <alias>` entries; each alias holds one or more `Path` clips â€” settings are per-`Path` children (`Volume`/`Pitch`/`Loop`/`Relative`/`ReferenceDistance`/`MaxDistance`/`Rolloff`) â€” plus an optional `Select` mode `Single`/`Random`/`RandomNoRepeat`/`Cycle` (`EAudioSelect`) that picks a clip per trigger; `trigger(entity, alias, overrides)` plays one, playing spatial sounds follow the entity unless a position override pinned them, buffers World-cached per path; demo: physicsCube.pre), `ZoneComponent`/`CullingComponent` (mostly placeholders)
* Script glue lives here, not in Script: `Globals::scriptContext` (the `ScriptContext` ABI singleton + host thunks), `ScriptEventManager` (`Globals::scriptEvents`) â€” global named events (`fireEvent("W Down")`), scripts subscribe by declaring On Event entries, mapping rebuilt on script reload. Its `initialize()` must run from main before any scripted entity spawns (cross-library global construction order). Deferred root-list mutations from scripts (`Globals::scriptRootAdditions/scriptRootRemovals/scriptDestroyRequests`) are drained by the App main loop each frame
* `dispatchPhysicsContactEvents()` (called from main after `physics.update`) maps contact/sensor events onto entities: invokes `PhysicsComponent::onContact` and fires the script's `OnPhysicsEvent` entry point (`ScriptComponent::firePhysicsEvent`)

## Script
* Visual scripts compile to real DLLs: `ScriptHost` (`Globals::scriptHost`, depends only on Core) compiles a .scr with the installed MSVC toolchain (finds vcvars64.bat via vswhere, then `cl /LD /std:c++20 /MD /Od /Zi /DSCRIPT_BUILD`) into a self-contained DLL under `Assets/Local/Scripts/`, caches by path, hot-reloads on demand (`getOrLoad(path, forceRecompile)`) â€” F6 or the Script panel's Compile & Run; live entities hot-swap. On failure the previous build is kept; superseded PDBs are garbage-collected around the debugger
* A .scr file IS the generated C++ (`#include "ScriptAPI.h"`, entry points `OnSpawn`/`Update`/`OnDestroy`/`OnEvent`/`OnPhysicsEvent` + `ScriptEventCount`/`ScriptEventName`/`ScriptDataSize`) with the node graph appended as `//@graph` / `//@node` metadata comments â€” the NodeEditor regenerates the code from the graph, and the file is also hand-editable C++
* `ScriptAPI.h` is the hostâ†”DLL ABI: C linkage, PODs/glm/raw pointers only (no STL, no engine types). The `ScriptContext` function table is APPEND-ONLY (cached DLLs compiled against older layouts must keep working). Scripts see a layout-compatible `Entity` MIRROR struct (pos/scale/rot/parent only â€” the layout past `displayName` is engine-internal); the engine static_asserts the offsets in ScriptContext.cpp so drift fails the build. Everything else goes through `ctx->entity*` functions (name, children, animation params, physics velocity/impulse/raycast, spawn/destroy, events)
* `ScriptModule`: the loaded DLL's entry-point pointers + declared event names; don't cache the function pointers across reloads

## Physics
* Box3D (Erin Catto's 3D engine, C API) wrapped in `module Physics`; depends only on Core, box3d is linked PRIVATE and never leaks into the public surface (body/joint/mesh ids stored as integer bits)
* `Globals::physics` (`PhysicsWorld`): `initialize()` once, `update(deltaSec)` per frame (fixed-step accumulator; gravity/paused/interpolate/timescale/substeps/Hz are Tweak vars under Physics/World), `createBody(PhysicsBodyDesc, span<PhysicsShape>)` (Box/Sphere/Capsule/Hull/Mesh; the desc transform's scale is baked into shape dims), `castRayClosest()`
* `PhysicsBody`/`PhysicsJoint`/`PhysicsMesh`: RAII movable handles, like RenderNode. Joints (`createDistance/Revolute/Spherical/WeldJoint`) take world-space anchors/axes; `staticBody()` anchors joints to the world. `PhysicsMesh` (triangle BVH) is referenced by Mesh shapes and must outlive them; Hull shapes clone at creation. Mesh colliders only collide on static bodies
* `PhysicsComponent` in Entity: `Component Physics` in .pre files (`Body` Static/Kinematic/Dynamic, `Shape` Box/Sphere/Capsule/Hull/Mesh, `HalfExtents`/`Radius`/`HalfHeight`/`Offset`/`Density`/`Friction`/`Restitution`/`MaxHullVertices`/`Sensor`/`ContactEvents`, see `Assets/Entities/Debug/physicsCube.pre`). Hull/Mesh shapes source geometry from the sibling render mesh: a `CollisionSource` snapshot (positions/indices + node tree) is captured from the same `ISceneData` the container loads from â€” one import, not two (re-import only as fallback when the container was loaded before physics needed it). Meshes/nodes named `Col_*` are collision proxies: physics collides with them instead of the same-named render mesh (`Col_Wall` replaces `Wall`; meshes without a proxy collide as themselves) and the renderer never instances them (skipped in `ObjectContainer::initializeNodes`). Dynamic bodies write the simulated pose back into the entity's local transform each update, interpolated between fixed steps; kinematic/static bodies follow the entity when it's moved (gizmo/scripts). Sponza has a static mesh collider (sponza.pre)
* Contact/sensor events: shapes opt in via `ContactEvents`/`Sensor`; `PhysicsWorld::update` drains them per step into `getContactEvents()` (body userData pairs, plus a packed box3d `contactId` â€” 0 for sensor events); `dispatchPhysicsContactEvents()` (Entity, called from main loop) invokes `PhysicsComponent::onContact` and fires the script's `OnPhysicsEvent` entry point (the "On Physics Event" node â€” a fixed entry point like OnSpawn/Update, not a named-entry node like On Event) with the other entity, begin/sensor bools, and the contactId; `PhysicsWorld::getContactPoint(contactId, ...)` (script node "Get Contact Point") resolves it to a world-space hit position/normal, valid only until the next `physics.update()`
* Collision masking via named layers: `PhysicsLayers::bit(name)` allocates one of 64 category bits on first use ("Default" = bit 0); `PhysicsShape` carries `categoryBits`/`maskBits`/`groupIndex`. In .pre: `Layer Debris` + `CollidesWith Default, Player` (or `All`/`None`) + optional `Group <int>`; two shapes collide when each one's mask contains the other's category. `castRayClosest` takes an optional layer mask (default all). Demo: physicsSphere.pre spheres pass through each other

## Particle
* GPU particle systems + projected decals (links Core + RendererVK + File). The renderer side lives in RendererVK (`ParticlePipeline`/`DecalPipeline` under Private/Pipeline, GPU structs + MAX_* constants in Layout.ixx, injected into shader compiles); this library is the gameplay-facing layer: `Globals::particleSystem` (`ParticleSystem`) — `initialize()` from main before world spawns, `update(renderer, deltaSec)` per frame AFTER `world.update` (turns emitter rates/bursts into GPU spawn requests, re-uploads every emitter's config — so live .pfx/param edits retroactively drive spawned particles — ages + submits decals)
* Particles simulate ON THE GPU (`particle_begin/emit/sim.cs.glsl`): one persistent device-local pool (`MAX_PARTICLES` 256K x 48 B), dead-index stack, two alive lists ping-ponged by frame parity; all dispatches indirect (begin = pool reset or prepare, emit = CPU-written groups over the per-frame spawn map, sim = GPU-sized over survivors+spawns) and the OUT alive count IS the draw's instanceCount, so spawning/emitter changes never re-record. Sim: gravity/drag/value-noise turbulence + optional screen-space depth collision vs LAST frame's G-buffer; billboards draw in the scene-color pass (after decals, before fog apply): velocity stretch, flipbooks, per-particle GI-probe+sun lighting (LIT flag), soft depth fade, premultiplied blend with per-emitter additivity (0 = alpha smoke .. 1 = additive fire) in ONE pipeline. Tweaks under Particles/* (enabled, collision, time scale)

## Threading
* Fiber-based work-stealing job scheduler (depends only on Core; replaced the old unused coroutine `Core.JobSystem`). Jobs run ON Win32 fibers, so `wait()` inside a job PARKS the fiber and frees the worker to run other jobs; the fiber resumes on whichever worker services the resume queue — possibly a different thread, which is why the root CMakeLists compiles everything `/GT` (fiber-safe TLS): job code must not cache `thread_local` addresses across a wait
* `Globals::jobSystem` (`JobSystem`): `initialize()` MUST run on the main thread (registers it as the helper context: main-thread `wait()` runs jobs instead of blocking; unregistered threads block on the counter atomic). `shutdown()` from main before exit — joins workers while other globals still live. `submit(callable, priority, &counter, name)` / `submitDelayed(delaySec, ...)` (min-heap + dedicated timer thread) / `wait(counter)` / `parallelFor(begin, end, grain, func)` (shared atomic cursor, caller participates, safe to nest inside jobs). Callables store INLINE in the 128-byte `Job` (`Job::StorageSize` = 72; static_asserted — capture a pointer to bigger state). `JobCounter`: add-before-submit is internal to submit; decrement-to-zero resumes parked fibers, notifies blocked threads. Counters/data captured by reference must outlive the wait (fiber stacks make stack-locals safe when the same scope waits)
* Scheduling: per-context Chase-Lev steal deque (`Threading:StealDeque`, fixed capacity, owner LIFO / thieves FIFO) + 3 shared Vyukov MPMC priority rings (`Threading:MPMCQueue`) + resume queue + free-fiber/job-pool rings — everything preallocated in `initialize`, hot paths lock-free, zero runtime allocation. Worker pop order: resumed fibers, own deque, High/Normal/Low rings, steal (random victim). A worker's own continuations always go to its LOCAL deque regardless of priority (fork/join locality); priorities order the shared rings only. Idle workers spin briefly then sleep on an eventcount (`m_wakeEpoch` atomic wait/notify, Dekker-paired with `m_numSleepers`); on exhaustion (pool/queue full) submits execute INLINE with a debug assert — raise the `JobSystemDesc` capacities
* `Threading:JobSync`: `JobMutex` (fiber-parking mutex: contended lock inside a job parks the fiber, main helps, external threads block; barging, standard lifetime contract — use for long exclusive sections like V3 inference so waits hold a fiber, not a worker), `JobEvent` (manual-reset over `JobSystem::signal(counter)`, idempotent signal), `PerWorker<T>` (cacheline-aligned per-context slots via `getWorkerIndex()`/`getNumContexts()`, `.local()`/`.forEach()` — the substrate for per-worker command queues/dirty lists; never cache `.local()` or the index across a `wait()`)
* Job cost heuristic: graph nodes are wall-time EMA'd (`EJobFlag_Timed`, min 1µs so all-cheap graphs rank by chain depth; parked waits count — they occupy critical-path time) and each `run()` recomputes critical-path ranks in one reverse-index walk (edges are forward-only), sorts roots rank-descending and promotes Normal-priority nodes in the top rank quartile to the High ring (`Job::effectivePriority`; user High/Low respected; threshold = previous run's max). `getJobCostUs/getJobRankUs(JobId)` = free per-node profiling. `parallelFor(begin, end, JobCost&, fn)` auto-grains: caller-owned `JobCost` tracker (seed = expected ns/item) targets `JobSystemDesc::parallelForTargetChunkNs` (25µs) chunks, runner timings feed back per call. Ad-hoc submits are deliberately untimed (two clock reads on ~100ns jobs); `Job::StorageSize` is now 64 bytes
* `JobGraph` + `JobResource` (`Threading:JobGraph`): declared-access DAG. `graph.addJob(name, fn).reads(resA).writes(resB).after(id).priority(p)` derives hazard edges render-graph-style in declaration order (reader <- last writer; writer <- readers since last write); same-resource reads run in parallel, and the schedule itself is the synchronization — no locks anywhere. Edges always point earlier->later declared, so cycles are impossible by construction. `compile()` once (dedupe + CSR successor arrays + initial dependency counts + roots), then `run()`/`wait()`/`runAndWait()` per frame: run only resets the per-job atomics and pushes roots — zero allocation, re-runnable. `JobResource` is a process-unique id created next to the data it names (non-copyable, like Tweak registration)
* Stress harness (`Globals::jobSystemStress`, Threading/Stress tweaks, `update()` from the main loop): benchmark modes (empty-job throughput, parallelFor, a 128-job/16-resource hazard-CHECKED random graph, fiber park/resume) publishing ms + jobs/ms, "Run self test" one-shot correctness pass (sums, coverage, hazard ordering, fiber waits, delayed jobs) into the log, scheduler rates under Threading/Stats
* Invariants (each guards a race that was actually hit; torture-tested via a temp console target, the way to re-verify changes here): a waiter may FREE the counter the instant it observes completion, so `isDone` is "the FULL value is zero", fiber-wait registrations ride the count atomic's high 8 bits (`JobCounter::WaiterInc`) keeping the value nonzero until their owner removes them, the zero transition claims the wait list with ONE `exchange` of a taken-sentinel (late registrants see the sentinel - strictly after that exchange - and back their own bit out), and the only post-zero touch is an address-only `notify_all`. A parked fiber's wait node lives on its stack (signaler reads `next` before resuming, spins on `switchDone` until the fiber fully switched out); `runFiber` reads the fiber state BEFORE publishing `switchDone` (the fiber may finish elsewhere and be reused); recycling freelists are LIFO tagged-index stacks, NOT rings (a preempted ring pop makes pushes see phantom "full"/"empty" - this exhausted the job pool in practice); inline helping in `wait()` is depth-capped (a helped job that waits helps again - unbounded recursion). `Job` is 128 bytes / two cache lines (static_asserted). Don't hold a fiber wait across `shutdown()` (asserted)

## Procedural
* The procedural world layer: diffusion-model terrain, FFT ocean, object scattering, terrain physics, and the camera-centered world-data bakes they share. Sits above the renderer (links RendererVK + File + Spatial + Physics; onnxruntime is PRIVATE and never leaks, like box3d in Physics). Not entity-based and no `Globals` here: main.cpp owns `TerrainStreamer terrain` / `TerrainCollider terrainCollider` / `OceanGenerator ocean` / `ScatterSystem scatter` and updates them per frame in that order (after entity updates, before `present`), wiring `ocean.swellTravelAngle()` -> `terrain.setFlowWindAngle` before `terrain.update`, then `terrainCollider.update(camera.position, terrain.activeClimateMaps())`, `ocean.update(renderer, camera, terrain.activeTerrainData(), terrain.seaLevel())` and `scatter.update(..., terrain.activeClimateMaps())`
* `ITerrainSampler` (`:TerrainSampler`): THE terrain field interface everything consumes — pure virtual, pure functions of world (x, z), seeded/thread-safe/world-continuous, held as `shared_ptr<const>` (workers finish against old maps across config rebuilds). `samplePoint`/`sampleGrid` fill a `TerrainPoint` (height, water level, altitude, temperature + SEA-LEVEL baseline, humidity, fog, flow angle) in ONE evaluation — wide-area consumers must use `sampleGrid` (hoists per-point overhead; for V3 one lock per tile instead of per texel); single-field samplers exist for point consumers. `ESampleDetail::Coarse` lets km-scale bakes read a cheap level. One `lapseRatePerMetre()` for the whole world (temperature = baseline + lapse x height; per-texel rates measured redundant)
* `TerrainGenV3` (`:GeneratorV3`, implementation under `Private/Diffusion/`): the terrain generator — an ONNX diffusion pipeline port (ONNX Runtime + DirectML, 2.28 GB of models downloaded/loaded once per process, then RESEEDED never reloaded, so sampler rebuilds are cheap). Generates 256x256 TILES (7.68 km at the native 30 m/px; `metersPerPixel` is a UNIFORM world scale) behind the sampler interface: a point read is a bilinear tile fetch, a cold tile takes ~1.5 s with inference serialized by a fiber-parking `JobMutex` (tile store is not thread-safe; all procedural jobs call in — their waits park fibers, not workers), and concurrent requesters of the same tile park on a per-tile `JobEvent` (one generation serves all). Sub-30 m relief is slope-masked procedural noise on top (`:Noise` `NoiseField`, fbmEroded). `isReady()` gates everything until models are loaded (else a flat world bakes into caches). Diffusion .cpp TUs compile `/fp:precise` + `/wd5050` — load-bearing; the why and the .ixx-declarations-only rule it forces are documented in Code/Procedural/CMakeLists.txt
* `TerrainStreamer` (Tweaks "Terrain*"): render-only chunk streaming — a bounded LOD ring of chunks around the camera, generated on up to "Gen jobs" self-continuing Low-priority pump jobs (default 2; `kickPump(n)` CAS-claims slots + exit-recheck protocol; nearest-first pick at dequeue kept; warm-tile mesh builds overlap cold V3 waits since those park fibers; `createMeshScene` runs IN the pump — audited pure — so main only does ObjectContainer::initialize; ring enqueue scan gated on ring/pending change; splat bake = one Low job wrapping a parallelFor). ScatterSystem uses the same multi-pump protocol ("Scatter/Gen jobs") (`generateChunk` from `:TerrainGenerator`: world-space sampling so chunk/LOD boundaries agree, skirts, seabed included) and bridged to ObjectContainer/RenderNode + SpatialEntry on the main thread; chunks leaving the ring free all GPU residency (`~ObjectContainer`). Owns THE world datum + rules other systems must agree with, handed out not duplicated: `seaLevel()`, `activeWaterReach()`, `activeFlowField()`, `activeClimateMaps()`. Also bakes the shared TERRAIN-DATA MAP (2 cascades near/far, RGBA32F: height, water level, bit-packed fog|flow|temp|humidity, macro altitude -> `Renderer::setFogTerrainHeightMap`; consumed by volumetric fog terrain-follow, the ocean's water depth/level (GPU via terrain_height.inc.glsl + a CPU copy via `activeTerrainData()` for buoyancy/wind steering), terrain coloring) and the terrain splat textures (BC-baked to Assets/Local/TerrainTex on a background thread at startup, climate-box picked)
* `HeightMapBaker` + `applyWaterReach` + `applyFlowField` (`:HeightMapBaker`): the async snapshot bake driver both maps use — one Low-priority job in flight (JobCounter-polled, params boxed past the 64-byte job capture), re-bakes on camera drift > range/4 or when sampler identity/ranges/rules change (configs carry `operator==` so the tweaks stay live), center snapped to the coarsest texel lattice. `WaterReach`: chamfer distance-to-ocean sinks the baked water level under ground the ocean cannot reach (V3 reports sea level planet-wide; this is what keeps swash/beaches off inland hollows). `FlowField`: 8-bit per-texel flow direction (0 = none, 1..255 = angle/2pi) — toward nearest land through the surf zone (dead-reckoning feature transform + easing back to the swell heading offshore + mask-weighted vector-sum box blur so beach downhill never cancels it), downhill everywhere else; baked into BOTH maps' flow bits
* `OceanGenerator` (Tweaks "Ocean*"): CPU side of the FFT/Tessendorf ocean — a camera-following geometry clipmap (fixed cell size per ring so mips don't morph with camera motion, CDLOD edge morph, ONE mesh/node; a "Horizon band" quad ring extends the outermost ring's morphed lattice to the camera far plane so the sea always meets the horizon) over `RendererVKLayout::EPipelineIndex::Ocean`; the simulation (spectrum -> IFFT -> displacement/gradient maps, re-evaluated every frame so every param is live) is `OceanSimulationPipeline` in RendererVK. Bakes NOTHING itself: shoaling, surf, swash and the VS land cull all read the streamer's shared terrain-data map (GPU: terrain_height.inc.glsl cascades; CPU: the `activeTerrainData()` snapshot adopted each update). `sampleWaterHeight` = CPU buoyancy field from the GPU displacement readback (App wires it into `PhysicsWorld::setWaterSurface`); its `sampleDisplacement` is a full CPU MIRROR of `oceanSampleDisplacement` (ocean_wave.inc.glsl) — shoal fades, swash run-up/backflow/drawdown, waterline floor, and the `swashReach` estimate Renderer.cpp packs into the UBO. "Ocean/RT" tweaks budget the per-pixel scene rays (refraction range = underwater visibility, reflection range + roughness cutoff, and a camera-distance ray cutoff that falls back to the analytic bottom/sky). The shader is what you see and the mirror is what floats on it, so a change to either without the other is a silent bug (bodies sinking through drawn waves / floating on dry sand); returns -FLT_MAX only past the run-up band, and inside it the drawdown floor sinks the surface under the sand, which is what beaches bodies. WIND CONVENTION: the spectrum's dominant term travels AGAINST `windDirection` — the swell heading is `swellTravelAngle()` = wind angle + pi; `steeredWindAngle` turns the SIM wind toward the baked shore flow around the camera (rate-limited slew, votes from the CPU shore copy) so waves roll inland — per-pixel domain rotation is disabled in ocean_wave.inc.glsl (creases; see the comment there before reviving it)
* `TerrainCollider` (Tweaks "Terrain/Collision"): terrain physics — a camera-centered ring of small static triangle-mesh collider tiles (default 32 m tiles within 96 m, 1 m sample spacing = the render LOD0 lattice, generateChunk's triangulation, so bodies rest exactly on the drawn surface). Never touches the render chunks (LOD0 is ~500k tris; colliders only need the ground near dynamic bodies): each tile is sampleGrid'd + BVH-built (`createCollisionMesh` is standalone/thread-safe) on ONE in-flight Low-priority job (JobCounter-polled), nearest-first; the main thread only creates/destroys the static bodies (layer "Terrain"). Tiles clear on sampler identity change / disable, evict with half-a-tile hysteresis. Verify with Physics/Debug/Draw colliders
* `ScatterSystem` (Tweaks "Scatter"): trees/rocks/grass over the terrain — config is two tables in Scattering.cpp (`SCATTER_ASSETS`: .oc-referenced models with footprint/scale/slope settings; `SCATTER_RULES`: climate ATTRACTORS in real units (C, mm/yr) with Gaussian density falloff — no biome enum). Cell placements computed deterministically on a Low-priority pump job (same claim protocol as TerrainStreamer) from the SAME sampler the terrain renders (footprint dart-throw, cluster noise, slope/altitude bands), bridged to RenderNodes; per-rule `viewDistance` spawns/despawns instance groups without regenerating cells; everything regenerates when the sampler identity changes

## Network
* Winsock UDP/TCP + a reliable-UDP game protocol (depends only on Core; Ws2_32 + Bcrypt linked PRIVATE). Module partitions `:Address`, `:Serialization`, `:Socket`, `:Crypto` (internal, nothing exported), `:Reliable`, re-exported from `Public/Network.ixx`. IPv4 only, wire format is raw little-endian bytes â€” no versioning/compat, both ends must match (`protocolId`)
* `:Serialization`: `NetWriter`/`NetReader` over a caller-provided byte span â€” trivially-copyable `write<T>`/`read<T>`, varint (LEB128 + zigzag), strings, unorm/snorm/`writeQuantized<UInt>(v, min, max)` float quantization; overflow-safe (check `overflowed()`), reads are zero-copy views
* `:Socket`: non-blocking RAII wrappers â€” `UdpSocket` (big SO_RCVBUF, SIO_UDP_CONNRESET disabled, optional broadcast), `TcpSocket`/`TcpListener` (TCP_NODELAY, `poll()` resolves connects), `TcpMessageStream` (uint32-length-framed messages with partial send/recv buffering), `netResolveHost` (blocking DNS). Winsock initialized lazily on first open
* `:Reliable`: `NetHost` â€” symmetric client/server/p2p over one `UdpSocket` (two hosts `connect()`ing to each other resolve the simultaneous handshake, tie-broken on connect salts). Poll model like the rest of the engine: queue with `send(peer, bytes, ENetDelivery, channel)`, per-frame `update(dt)` pumps/retransmits/flushes (messages batched into â‰¤`maxPacketSize` packets), drain `takeEvents()` (Connected/Disconnected/Message). Deliveries: Unreliable, UnreliableSequenced (stale-dropped), Reliable (ordered + guaranteed, acked via packet seq + 32-bit ack bitfield piggybacked on every packet, RTT-based retransmit, fragments transparently up to `maxMessageSize`) â€” on up to 8 independent channels (index rides in the message kind byte, zero wire cost) so bulk transfers don't head-of-line-block other reliable streams. 4-way anti-spoof challenge handshake (responder is stateless until the challenge round-trips: `serverSalt = HMAC(hostSecret, addr|salt|flags)`); `NetHostConfig::encrypt` adds ephemeral ECDH P-256 key agreement in the handshake + AES-128-GCM per payload packet (+16B tag, nonce = sender role + implicit 64-bit packet counter; Windows CNG, hardware AES) â€” secure against spoofing/eavesdropping/tampering, not an active MITM (unauthenticated keys). Keepalives/timeouts, per-peer RTT/loss stats, and live-editable link simulation (`simPacketLoss`/`simLatencyMs`/`simJitterMs`) in `NetHostConfig`

# Asset files (all text formats parsed via `AssetParser`, registered by `AssetRegistry`)
* `.oc` (ObjectContainer): a model source + import options (`Path`, `MergeNodes`, `PreTransformVertices`, `DecimationFactor` (< 1 meshopt-simplifies base geometry at cook time; cache-only, so skinned/procedural containers never decimate), `MaterialOverrides` with `PipelineIdx`/`ExcludeFromRayTracing`/texture overrides) plus named `StaticMesh`/`SkinnedMesh` spawnable entries (`Node` path within the model; SkinnedMesh adds `Type Humanoid/Generic` + optional bone mapping). Skinned containers MUST keep `PreTransformVertices false`
* `.pre` (Prefab): a named entity (hierarchy) â€” `Prefab <name>` root with `Component Render/Animator/Script/Physics/Audio/Scene` children; `Component Scene` nests child `Prefab` references. `Component Audio` nests `Sound <alias>` entries, each with an optional `Select` and one or more `Path <file>` clips whose own child lines are that clip's playback settings. Render references a spawnable (`StaticMesh <name>`) or `ObjectContainer <name>` + `Node`, plus `Position/Rotation/Scale`. Spawn by prefab name (`world.spawn("physicsCube", ...)`) or file path (`world.spawnAssetFile`)
* `.anm`: `Animation <name>` clip entries â€” source `ObjectContainer` (fbx), `Loop`, `Skip` (channels), `Event <name> <normalizedTime>` notifies (fired into the entity's script/animator `onEvent`)
* `.apl`: `Animator <name>` graph â€” `Parameter`s (Float/Bool/Trigger), `Clip` bindings, `BlendSpace1D` with `Sample`s, `StateMachine` (states with `Play`/`SpeedParam`/`SpeedScale`, `Transition`/`AnyTransition` with `Condition`/`ExitTime`/`Fade`)
* `.scr`: visual script = generated C++ + `//@graph` node metadata (see Script)
* `Assets/Local/` is generated output (compiled SPIR-V + shader source dumps for Aftermath, compiled script DLLs/PDBs, cooked scene caches `Cooked/*.vsc` + converted `.dds`) â€” never hand-edit
