# Project Description
A modern c++ Game Engine codebase. Windows/MSVC only. The `App` executable is the testbed: it spawns `Entities/SponzaScene.pre` (Sponza + skysphere prefabs, Sponza with a static mesh collider) and `Entities/character.pre` (skinned, animated, scripted character) through the `World`, runs an ImGui editor UI (scene hierarchy, properties, asset browser, visual-script node editor, tweaks) around the viewport, and drives a transform gizmo. VR is initialized with `EVr::DISABLED` in main.cpp ├óŌé¼ŌĆØ flip to `ENABLED` to run on an OpenXR headset. A Bistro scene (`Entities/BistroScene.pre`) exists as an alternative test scene.

## App testbed key bindings
* F5 = shader hot-reload, F6 = recompile + hot-swap the current visual script
* T/R/G = gizmo translate/rotate/scale
* P = GI probe debug cubes, O = cycle probe debug mode, L = aim sun along camera
* 1 = clear spawned lights, 2/3 = point lights, 4 = spot, 5 = area, 6 = tube, 7 = 100 random point lights
* 8/9 = throw physics cube/sphere (with a positional audio blip), 0 = hang a physics chain (spherical joints)
* WASD/LShift also fire global script events ("W Down"/"W Up", ...) via `Globals::scriptEvents`; Ctrl+C/V = script node copy/paste

## Style
* Using exclusively c++20 modules instead of headers. (.ixx instead of .h)
	* Most libraries use module partitions: `export module RendererVK:Renderer;` in the interface .ixx, `module RendererVK;` in the implementation .cpp; public surface re-exported from `Public/<Lib>.ixx` (`RendererVK`, `Entity`, `Animation`, `File`, `Input` all do this)
	* `Core` and `UI` use dotted module names instead (`Core.Tweaks`, `UI.NodeEditor.Scene`, one module per .ixx)
	* `Public/` folder .ixx's are importable from outside the library; `Private/` is internal
	* The `Core` module re-exports std headers as header units ├óŌé¼ŌĆØ if you need a std header that isn't available, add it to `Code/Core/Public/Core.ixx`
	* `Core.fwd` / `*.fwd.ixx` modules hold forward declarations
	* One exception: `Script/Public/ScriptAPI.h` is a plain header ├óŌé¼ŌĆØ it's the ABI shared with runtime-compiled script DLLs and is `#include`d on both sides
* Modernest conventions
	* No new/delete, only memory safe types.
	* Raw pointers are non-owning
	* Minimal comments, only in exceptional cases where code intent is too difficult to communicate
	* Prioritize performance
* Engine singletons live in `export namespace Globals` inside the owning .ixx (`Globals::rendererVK`, `Globals::device`, `Globals::input`, `Globals::ui`, `Globals::world`, `Globals::physics`, `Globals::scriptHost`, `Globals::scriptEvents`, `Globals::assetRegistry`, `Globals::time`, `Globals::allocator`, ...). RendererVK globals use `#pragma init_seg` to control construction order ├óŌé¼ŌĆØ keep that when adding one. Globals in *different* libraries have no defined construction order relative to each other (see `ScriptEventManager::initialize` for the pattern when one global must register with another)
* int8/uint64-style typedefs from `Core`; glm via `Core.glm`; `assert` is compiled out in non-debug via forceinclude.h

## RendererVK
* Modern Vulkan renderer designed to only use modern features (minimum spec RTX 2000 series)
* Device Generated Commands (DGC) for everything
* Lights culled with a world space hash table indexing into grids with a cellcount based on distance to camera
* Global illumination using scrolling clipmap radiance cascades (ray traced probes)
* RTAO, TAA, PCSS cascades, volumetric fog, GPU compute skinning
* Everything upstream of composite is linear HDR: `EyeAdaptationPipeline` (log-luminance histogram + smoothed auto-exposure, compute) feeds `CompositePipeline` (fullscreen HDR├óŌĆĀŌĆÖdisplay pass into the swapchain before ImGui; tonemap operators: off/Reinhard/ACES/AgX)
* Frame order (see `recordCommandBuffers()` / `present()`): skinning compute ├óŌĆĀŌĆÖ indirect cull ├óŌĆĀŌĆÖ light grid ├óŌĆĀŌĆÖ shadow cull + draw ├óŌĆĀŌĆÖ static mesh/G-buffer ├óŌĆĀŌĆÖ GI probes ├óŌĆĀŌĆÖ RTAO ├óŌĆĀŌĆÖ volumetric fog ├óŌĆĀŌĆÖ TAA ├óŌĆĀŌĆÖ eye adaptation ├óŌĆĀŌĆÖ composite + ImGui
* Layer structure under `Private/`:
	* `Objects/` ├óŌé¼ŌĆØ thin Vulkan object wrappers (Device, SwapChain, Buffer, Pipeline, AccelerationStructure, GBuffer, SceneColor, ShadowMap, ...)
	* `Pipeline/` ├óŌé¼ŌĆØ one class per render pass/feature (GBuffer, GIProbe, RTAO, TAA, VolumetricFog, EyeAdaptation, Composite, Skinning, light grid + indirect/shadow culling compute, shadows). Each registers its own Tweak vars
	* `Data/` ├óŌé¼ŌĆØ MeshDataManager, TextureManager, StagingManager, ShaderDatabase (Aftermath)
	* `Renderer.ixx/.cpp` ├óŌé¼ŌĆØ orchestrates everything; per-pass `record*()` methods called from `recordCommandBuffers()`
	* `OpenXRSession.ixx` ├óŌé¼ŌĆØ VR: OpenXR session/swapchain wrapper (`Globals::openXR`, implements `IVrSession`). Two-phase init around Vulkan device creation (runtime dictates instance/device extensions + physical device); stereo rendering targets a 2-layer multiview swapchain; in VR the editor viewport sub-rect is ignored (full-extent render). `Renderer::initialize` takes `EVr::ENABLED/DISABLED`
* Scene objects: an `ObjectContainer` wraps one loaded `ISceneData` (meshes/materials/textures, optional `MaterialOverrides`); `container.spawnNodeForIdx()` returns a `RenderNode` instance handle (RAII, move-only), `spawnSkinnedNode()` the GPU-skinned variant (an `AnimatorComponent` feeds its bone palette via `allocateSkinningPalette`/`setSkinningPalette`). Destroying a `RenderNode` recycles its resources without GPU sync (all per-frame data is re-uploaded anyway): transform slots go on a free list, a skinned node's whole allocation bundle (`Renderer::SkinnedInstanceBundle`) is parked in place and reused by the next `spawnSkinnedNode` of the same container, and rebased sub-node instance offsets are uploaded once per NodeSpawnIdx and shared. Container-level resources still live for the app's lifetime (`~ObjectContainer` TODO). Lights are added per-frame via `renderer.addPointLight/addSpotLight/addAreaLight/addLightInfo/...`
* Per-frame instance flow: `renderNode(node, passMask)` writes the node's instances + a per-node pass mask (`PASS_MAIN/SHADOW/GI`, Layout.ixx = Spatial's pass bits) into mapped per-frame buffers; the main indirect cull, shadow cull, and GI TLAS-instance writer each early-out on their bit (TLAS also range-bounds by `RT/GI / TLAS Range`). Render node transforms upload SPARSELY: write only via `RenderNode::setTransform` (change-detected, per-frame-in-flight dirty lists drained in `present()`) - a mutable getTransform would bypass the tracking
* Texture mip streaming (`TextureStreamer`, "Streaming" tweaks): DDS textures load tail-only (mips <= 128px) and stream mips in/out of a fixed VRAM budget by camera priority (projected-size pass in `renderNode`). Residency changes recreate the image and rewrite that slot in all 5 bindless texture arrays (UPDATE_AFTER_BIND, applied per frame slot in `recordCommandBuffers` after the fence wait); disk reads run on a worker thread; old images retire after NUM_FRAMES_IN_FLIGHT. Non-DDS/procedural/embedded textures are pinned at full res
* Mesh data streaming (`MeshStreamer`, "Streaming" tweaks): cooked scenes register each source mesh + its generated LOD levels as a "mesh set" carrying the byte ranges its data occupies in the .vsc cache; the vertex/index mega-buffers are free-listed (`MeshDataManager`, BitRangeAllocator in element-aligned buckets). When resident bytes exceed "Mesh budget (MB)", the least-recently-referenced sets unseen for "Mesh cold frames" evict: their MeshInfos get indexCount 0 (DGC draws/shadow/TLAS writes become no-ops with no re-record; static BLASes persist - they are snapshots of identical geometry) and their ranges return to the free list after NUM_FRAMES_IN_FLIGHT. An evicted set referenced again re-streams from the .vsc on a worker thread (attributes re-interleaved to MeshVertex) and restores its MeshInfos at fresh offsets; the LOD selector renders the nearest resident level meanwhile. BLASes evict and rebuild with their set, and a LOD chain shares ONE BLAS at its "RT/BLAS LOD level" (alias table in AccelerationStructure; the TLAS-instance writer packs the aliased RT meshIdx into each instance's sbtOffset, which hit shaders use for geometry fetches while materials stay per-instance); static BLASes are copy-compacted a few frames after build ("RT/BLAS compaction", fence-timed size queries, ~30-50% back) Only cooked static meshes stream - skinned/procedural/direct-import data is pinned. NOTE: with Spatial culling Off everything is referenced every frame, so only despawned containers and Col_ proxies ever go cold
* Mesh LODs ("LOD" tweaks): meshes/nodes named `LodN_<name>` form authored chains grouped under `<name>` (a `Lod0_X` node path also resolves as `X`; level>0 never instanced, like `Col_`); static meshes without authored LODs get meshopt-generated index-only chains (shared vertices, "Generated reduction" of the indices per level; pre-baked by the scene cache when the source was cooked, generated at load otherwise). Selection runs per instance ON THE GPU (mesh_lod.inc.glsl in the main + shadow cull shaders; params via UBO lodParams, so tweaks are live): instances are pushed referencing LOD0 and the cull redirects the draw to the selected level's MeshInfo ŌĆö the CPU sizes every chain member's instance bucket to the chain's full count so any split fits, hysteresis state lives in a per-instance GPU slot range (RenderNode::m_lodStateBase + per-frame node bias buffer; races/garbage benign ŌĆö clamped into the frame's valid band), the shadow cull selects stateless at 4x/+2 coarser, GI needs no selection (all levels alias one chain BLAS), and per-level pick counts read back to Stats a few frames late. Selection is screen-space error: generated chains carry per-level meshopt simplify errors (mesh-local units, scaled by instance scale, projected to pixels) and use the coarsest level under the "Max error (px)" budget ŌĆö object size needs no separate tuning; authored chains without error data fall back to the projected-diameter/"Full-res pixels" metric. renderNode keeps every level of a referenced chain warm in the mesh streamer (once-per-group frame stamp), and the cull falls back to the nearest resident level (indexCount 0 = evicted) while a re-stream lands. Skinned meshes get bind-pose-generated chains too (index-only over the per-instance deformed output regions, which preserve source vertex ordering; per-instance MeshLodGroups registered at spawnSkinnedNode, recycled with the bundle; the per-frame skinned BLAS builds at the chain's "RT/BLAS LOD level"). Skinned output MeshInfos are excluded from static BLAS builds/compaction (uninitialized until the skinning compute runs; their addresses are owned per frame slot by the skinned rebuild)
* Try to record CommandBuffers once (not per frame) whenever possible; call `setHaveToRecordCommandBuffers()` after changes that invalidate them
* Shaders are GLSL in `Assets/Shaders/`, compiled at runtime with glslang (so shader edits don't need a rebuild ├óŌé¼ŌĆØ F5 in the app calls `reloadShaders()`). `*.inc.glsl` files are includes; `shared.inc.glsl`/`ubo.inc.glsl` define structs that must stay in sync with `Private/Layout.ixx` (`RendererVKLayout`). Compiled SPIR-V + source dumps land in `Assets/Local/` for Aftermath GPU crash analysis

## Entity
* The ECS. Inline component layout: an `Entity` (pos/scale/rot, parent, displayName, spawn template, refcount, `typeBits`) and its components live in ONE allocation ├óŌé¼ŌĆØ `typeBits` is a bitmask over the 8 component types (fixed order: Scene, Zone, Cull, Render, Animator, Script, Physics, Audio; `EComponentID` in `EntityDef.ixx`), and `getComponent<T>(entity)` / `hasComponent<T>` compute the byte offset from it at compile-time-known sizes (`Component.ixx`). Adding a component type means: new struct with `getId()`, entry in `EComponentID` + `inlineSizes` + `componentTypeName` + create/destroy/serialize switches in Entity.cpp, bump `MaxInlineComponentTypes`
* `EntityPtr`: intrusive refcounted handle (atomic on `refCount`); `Entity::create(template, transform)` placement-news into `Globals::entityAllocator`. Entities form a tree via `SceneComponent::children` + `Entity::parent`; `reparentEntity` moves ownership
* `Entity::update()` ├óŌĆĀŌĆÖ `updateTree`: per entity runs ScriptComponent ├óŌĆĀŌĆÖ AnimatorComponent ├óŌĆĀŌĆÖ PhysicsComponent, composes the world transform, refreshes the RenderComponent's SpatialIndex entry and pushes its `RenderNode` to the renderer (gated on `SpatialIndex::isVisible` when Spatial/Culling Mode ├óŌĆ░┬ź SafeCull), recurses into scene children. Disabled `SceneComponent` prunes the subtree
* `World` (`Globals::world`): `spawn("prefabName", transform)`, `spawnAssetFile("Entities/x.pre", ...)`, `createEmptyEntity`. Builds and caches `EntitySpawnTemplate`s (archetype + per-component `SpawnInfo` recipes) from parsed assets; caches `ObjectContainer`s, collision sources/meshes, and retargeted animation clip sets so each source file is imported once. `reloadPrefabs`/`invalidatePrefab` for editing (retired templates stay alive for live entities)
* `AssetRegistry` (`Globals::assetRegistry`): scans `Assets/` at startup and registers everything by name ├óŌé¼ŌĆØ `ObjectContainer` + `StaticMesh`/`SkinnedMesh` spawnables (.oc), `Animation` clips (.anm), `Animator` graphs (.apl), `Prefab`s (.pre)
* Prefabs: `savePrefab(root, path)` serializes an entity tree back to .pre (per-component `serialize`/`writeRenderSpawnInfo`/...); spawned prefabs are locked instances (`EEntityFlag_PrefabInstance`, unpackable in the Scene panel); `prefabWouldCycle` guards recursive saves
* Components: `SceneComponent` (children + enabled), `RenderComponent` (RenderNode + local transform, static or skinned), `AnimatorComponent` (AnimationPlayer + AnimStateMachine from a .apl, ticks and pushes the skinning palette; gameplay via `stateMachine.setFloat/Bool/Trigger`, clip events via `onEvent`), `ScriptComponent` (runs a .scr each frame with the entity as `self`), `PhysicsComponent` (see Physics), `AudioComponent` (named triggerable sounds: `Component Audio` with `Sound <alias>` entries; each alias holds one or more `Path` clips ├óŌé¼ŌĆØ settings are per-`Path` children (`Volume`/`Pitch`/`Loop`/`Relative`/`ReferenceDistance`/`MaxDistance`/`Rolloff`) ├óŌé¼ŌĆØ plus an optional `Select` mode `Single`/`Random`/`RandomNoRepeat`/`Cycle` (`EAudioSelect`) that picks a clip per trigger; `trigger(entity, alias, overrides)` plays one, playing spatial sounds follow the entity unless a position override pinned them, buffers World-cached per path; demo: physicsCube.pre), `ZoneComponent`/`CullingComponent` (mostly placeholders)
* Script glue lives here, not in Script: `Globals::scriptContext` (the `ScriptContext` ABI singleton + host thunks), `ScriptEventManager` (`Globals::scriptEvents`) ├óŌé¼ŌĆØ global named events (`fireEvent("W Down")`), scripts subscribe by declaring On Event entries, mapping rebuilt on script reload. Its `initialize()` must run from main before any scripted entity spawns (cross-library global construction order). Deferred root-list mutations from scripts (`Globals::scriptRootAdditions/scriptRootRemovals/scriptDestroyRequests`) are drained by the App main loop each frame
* `dispatchPhysicsContactEvents()` (called from main after `physics.update`) maps contact/sensor events onto entities: invokes `PhysicsComponent::onContact` and fires the script's `OnPhysicsEvent` entry point (`ScriptComponent::firePhysicsEvent`)

## Script
* Visual scripts compile to real DLLs: `ScriptHost` (`Globals::scriptHost`, depends only on Core) compiles a .scr with the installed MSVC toolchain (finds vcvars64.bat via vswhere, then `cl /LD /std:c++20 /MD /Od /Zi /DSCRIPT_BUILD`) into a self-contained DLL under `Assets/Local/Scripts/`, caches by path, hot-reloads on demand (`getOrLoad(path, forceRecompile)`) ├óŌé¼ŌĆØ F6 or the Script panel's Compile & Run; live entities hot-swap. On failure the previous build is kept; superseded PDBs are garbage-collected around the debugger
* A .scr file IS the generated C++ (`#include "ScriptAPI.h"`, entry points `OnSpawn`/`Update`/`OnDestroy`/`OnEvent`/`OnPhysicsEvent` + `ScriptEventCount`/`ScriptEventName`/`ScriptDataSize`) with the node graph appended as `//@graph` / `//@node` metadata comments ├óŌé¼ŌĆØ the NodeEditor regenerates the code from the graph, and the file is also hand-editable C++
* `ScriptAPI.h` is the host├óŌĆĀŌĆØDLL ABI: C linkage, PODs/glm/raw pointers only (no STL, no engine types). The `ScriptContext` function table is APPEND-ONLY (cached DLLs compiled against older layouts must keep working). Scripts see a layout-compatible `Entity` MIRROR struct (pos/scale/rot/parent only ├óŌé¼ŌĆØ fields past `std::string displayName` differ under the debug CRT); the engine static_asserts the offsets in ScriptContext.cpp so drift fails the build. Everything else goes through `ctx->entity*` functions (name, children, animation params, physics velocity/impulse/raycast, spawn/destroy, events)
* `ScriptModule`: the loaded DLL's entry-point pointers + declared event names; don't cache the function pointers across reloads

## Physics
* Box3D (Erin Catto's 3D engine, C API) wrapped in `module Physics`; depends only on Core, box3d is linked PRIVATE and never leaks into the public surface (body/joint/mesh ids stored as integer bits)
* `Globals::physics` (`PhysicsWorld`): `initialize()` once, `update(deltaSec)` per frame (fixed-step accumulator; gravity/paused/interpolate/timescale/substeps/Hz are Tweak vars under Physics/World), `createBody(PhysicsBodyDesc, span<PhysicsShape>)` (Box/Sphere/Capsule/Hull/Mesh; the desc transform's scale is baked into shape dims), `castRayClosest()`
* `PhysicsBody`/`PhysicsJoint`/`PhysicsMesh`: RAII movable handles, like RenderNode. Joints (`createDistance/Revolute/Spherical/WeldJoint`) take world-space anchors/axes; `staticBody()` anchors joints to the world. `PhysicsMesh` (triangle BVH) is referenced by Mesh shapes and must outlive them; Hull shapes clone at creation. Mesh colliders only collide on static bodies
* `PhysicsComponent` in Entity: `Component Physics` in .pre files (`Body` Static/Kinematic/Dynamic, `Shape` Box/Sphere/Capsule/Hull/Mesh, `HalfExtents`/`Radius`/`HalfHeight`/`Offset`/`Density`/`Friction`/`Restitution`/`MaxHullVertices`/`Sensor`/`ContactEvents`, see `Assets/Entities/Debug/physicsCube.pre`). Hull/Mesh shapes source geometry from the sibling render mesh: a `CollisionSource` snapshot (positions/indices + node tree) is captured from the same `ISceneData` the container loads from ├óŌé¼ŌĆØ one import, not two (re-import only as fallback when the container was loaded before physics needed it). Meshes/nodes named `Col_*` are collision proxies: physics collides with them instead of the same-named render mesh (`Col_Wall` replaces `Wall`; meshes without a proxy collide as themselves) and the renderer never instances them (skipped in `ObjectContainer::initializeNodes`). Dynamic bodies write the simulated pose back into the entity's local transform each update, interpolated between fixed steps; kinematic/static bodies follow the entity when it's moved (gizmo/scripts). Sponza has a static mesh collider (sponza.pre)
* Contact/sensor events: shapes opt in via `ContactEvents`/`Sensor`; `PhysicsWorld::update` drains them per step into `getContactEvents()` (body userData pairs, plus a packed box3d `contactId` ├óŌé¼ŌĆØ 0 for sensor events); `dispatchPhysicsContactEvents()` (Entity, called from main loop) invokes `PhysicsComponent::onContact` and fires the script's `OnPhysicsEvent` entry point (the "On Physics Event" node ├óŌé¼ŌĆØ a fixed entry point like OnSpawn/Update, not a named-entry node like On Event) with the other entity, begin/sensor bools, and the contactId; `PhysicsWorld::getContactPoint(contactId, ...)` (script node "Get Contact Point") resolves it to a world-space hit position/normal, valid only until the next `physics.update()`
* Collision masking via named layers: `PhysicsLayers::bit(name)` allocates one of 64 category bits on first use ("Default" = bit 0); `PhysicsShape` carries `categoryBits`/`maskBits`/`groupIndex`. In .pre: `Layer Debris` + `CollidesWith Default, Player` (or `All`/`None`) + optional `Group <int>`; two shapes collide when each one's mask contains the other's category. `castRayClosest` takes an optional layer mask (default all). Demo: physicsSphere.pre spheres pass through each other

## Spatial
* High-performance spatial index for culling and position-based entity lookup, independent of the parent/child hierarchy (depends only on Core). Structure: an implicit 64-ary hierarchy over per-level hashed grids ├óŌé¼ŌĆØ 11 levels, cell size ├āŌĆö4 per level (finest = compile-time `SPATIAL_FINEST_CELL_SIZE`, default 2m ├óŌĆĀŌĆÖ world bound ├é┬▒2097km), BMI2 Morton keys (`Spatial:Morton`; a level-L key is the fine key >> 6L, parent = key >> 6), each `CellRecord` holds a 64-bit occupancy mask of its 4├āŌĆö4├āŌĆö4 children so queries descend with bit scans and never visit empty space. API takes dvec3 (quantized to an int64 lattice), per-entry data is SoA with cell-relative float positions, and all query math is reference-relative float ├óŌé¼ŌĆØ exact at planet scale
* `Globals::spatialIndex` (`SpatialIndex`): `initialize()` from main before world spawns; `registerEntry(pos, radius, userData, layerMask)` -> `SpatialHandle` (`SpatialEntry` = RAII wrapper); `updateEntry` per frame (same-cell rewrite is ~4 stores, cell changes are queued); `commitFrame()` once per frame (main loop, after `beginFrame`) applies queued link/move/unlink ops and sweeps empty cells. Queries are read-only between commits: `querySphere/queryAABB/queryFrustum/queryRay/queryNearest`. Two per-pass visibility stamp sets (`markVisibleSet(Main, frustum)` + `markVisibleSphere(Near, ball)`, `getPassMask`/`isVisible`; single consumer per pass: the render gate); frustum queries take camera-relative planes (`rebaseFrustum`/`inflateFrustum`) and an optional cell-granularity `IOcclusionTester`
* Entity integration: every `RenderComponent` auto-registers on spawn (bounds from `RenderNode::getWorldBounds`, skinned x `Skinned radius scale` tweak, layer `SpatialLayer_Render`); `updateTree` refreshes the entry and pushes a per-pass mask: Main-visible entities push `PASS_ALL`, Near-only (off-screen but within `Near radius`) push `PASS_SHADOW|PASS_GI` so the GPU shadow cull and TLAS keep them while the main-view cull skips them. `Spatial/Culling/Mode` tweak: Off (default) / StatsOnly / Cull (per-pass, the usable mode) / MainOnly (debug; visibly kills off-screen shadows/GI); `Freeze` stops re-stamping to inspect the culled set
* CPU software occlusion (`Spatial:Occlusion`, `Globals::occlusionBuffer`, Spatial/Occlusion tweaks, default off): the largest triangles of static mesh colliders (extracted in `World::getOrBuildCollisionMesh`, registered by `PhysicsComponent` once its world transform is synced, RAII `SpatialOccluder`) are rasterized camera-relative into a 256x144 CPU depth buffer + 8x8 max-depth blocks each frame; plugged into `markVisibleSet(Main)` as the `IOcclusionTester`, so occluded cells drop out of the MAIN pass only (shadows/GI keep them). Conservative toward visible: pixel-center coverage, farthest-depth blocks, near-plane crossers always visible
* Stress harness (`Globals::spatialStress`, Spatial/Stress tweaks): spawn/clear N synthetic entries on `SpatialLayer_Stress` (invisible to gameplay/render queries), churn % random-walk, timed sphere/frustum queries, brute-force verify delta; live index stats under Spatial/Stats
* Scripts: `ctx->spatialQueryRadius` / `ctx->spatialGetNearestEntity` (ABI appends), NodeEditor nodes "Get Nearest Entity" + "For Each Entity In Radius" (category Spatial)
* Static tier: entries unchanged for N frames (Spatial/Static tweaks) promote into per-level Morton-sorted SoA ranges (`StaticStore`, addressed by `CellRecord.staticStart/Count`) via budgeted per-level rebuilds (max one level per commit; pending promotions stay linked + queryable until consumed); any change demotes (store tombstone = negative radius, dropped at the next rebuild). Frustum queries test static ranges 8-wide (AVX2 `test8`); other testers iterate them scalar
* Planned (not yet built): JobSystem-parallel traversal (the coroutine JobScheduler has no consumers anywhere yet - prove it elsewhere first)

## Procedural
* The procedural world layer: diffusion-model terrain, FFT ocean, object scattering, and the camera-centered world-data bakes they share. Sits above the renderer (links RendererVK + File + Spatial; onnxruntime is PRIVATE and never leaks, like box3d in Physics). Not entity-based and no `Globals` here: main.cpp owns `TerrainStreamer terrain` / `OceanGenerator ocean` / `ScatterSystem scatter` and updates them per frame in that order (after entity updates, before `present`), wiring `ocean.swellTravelAngle()` -> `terrain.setFlowWindAngle` before `terrain.update`, then `ocean.update(renderer, camera, terrain.activeClimateMaps(), terrain.activeWaterReach(), terrain.seaLevel(), terrain.activeFlowField())` and `scatter.update(..., terrain.activeClimateMaps())`
* `ITerrainSampler` (`:TerrainSampler`): THE terrain field interface everything consumes ŌĆö pure virtual, pure functions of world (x, z), seeded/thread-safe/world-continuous, held as `shared_ptr<const>` (workers finish against old maps across config rebuilds). `samplePoint`/`sampleGrid` fill a `TerrainPoint` (height, water level, altitude, temperature + SEA-LEVEL baseline, humidity, fog, flow angle) in ONE evaluation ŌĆö wide-area consumers must use `sampleGrid` (hoists per-point overhead; for V3 one lock per tile instead of per texel); single-field samplers exist for point consumers. `ESampleDetail::Coarse` lets km-scale bakes read a cheap level. One `lapseRatePerMetre()` for the whole world (temperature = baseline + lapse x height; per-texel rates measured redundant)
* `TerrainGenV3` (`:GeneratorV3`, implementation under `Private/Diffusion/`): the terrain generator ŌĆö an ONNX diffusion pipeline port (ONNX Runtime + DirectML, 2.28 GB of models downloaded/loaded once per process, then RESEEDED never reloaded, so sampler rebuilds are cheap). Generates 256x256 TILES (7.68 km at the native 30 m/px; `metersPerPixel` is a UNIFORM world scale) behind the sampler interface: a point read is a bilinear tile fetch, a cold tile blocks ~1.5 s while inference runs on ONE serialized thread (tile store is not thread-safe; streamer + scatter + bake workers all call in). Sub-30 m relief is slope-masked procedural noise on top (`:Noise` `NoiseField`, fbmEroded). `isReady()` gates everything until models are loaded (else a flat world bakes into caches). Diffusion .cpp TUs compile `/fp:precise` + `/wd5050` ŌĆö load-bearing; the why and the .ixx-declarations-only rule it forces are documented in Code/Procedural/CMakeLists.txt
* `TerrainStreamer` (Tweaks "Terrain*"): render-only chunk streaming ŌĆö a bounded LOD ring of chunks around the camera, generated on a worker (`generateChunk` from `:TerrainGenerator`: world-space sampling so chunk/LOD boundaries agree, skirts, seabed included) and bridged to ObjectContainer/RenderNode + SpatialEntry on the main thread; chunks leaving the ring free all GPU residency (`~ObjectContainer`). Owns THE world datum + rules other systems must agree with, handed out not duplicated: `seaLevel()`, `activeWaterReach()`, `activeFlowField()`, `activeClimateMaps()`. Also bakes the shared TERRAIN-DATA MAP (2 cascades near/far, RGBA32F: height, water level, bit-packed fog|flow|temp|humidity, macro altitude -> `Renderer::setFogTerrainHeightMap`; consumed by volumetric fog terrain-follow, ocean fallback, terrain coloring ŌĆö shader side terrain_height.inc.glsl) and the terrain splat textures (BC-baked to Assets/Local/TerrainTex on a background thread at startup, climate-box picked)
* `HeightMapBaker` + `applyWaterReach` + `applyFlowField` (`:HeightMapBaker`): the async snapshot bake driver both maps use ŌĆö one `std::async` bake in flight, re-bakes on camera drift > range/4 or when sampler identity/ranges/rules change (configs carry `operator==` so the tweaks stay live), center snapped to the coarsest texel lattice. `WaterReach`: chamfer distance-to-ocean sinks the baked water level under ground the ocean cannot reach (V3 reports sea level planet-wide; this is what keeps swash/beaches off inland hollows). `FlowField`: 8-bit per-texel flow direction (0 = none, 1..255 = angle/2pi) ŌĆö toward nearest land through the surf zone (dead-reckoning feature transform + easing back to the swell heading offshore + mask-weighted vector-sum box blur so beach downhill never cancels it), downhill everywhere else; baked into BOTH maps' flow bits
* `OceanGenerator` (Tweaks "Ocean*"): CPU side of the FFT/Tessendorf ocean ŌĆö a camera-following geometry clipmap (fixed cell size per ring so mips don't morph with camera motion, CDLOD edge morph, ONE mesh/node) over `RendererVKLayout::EPipelineIndex::Ocean`; the simulation (spectrum -> IFFT -> displacement/gradient maps, re-evaluated every frame so every param is live) is `OceanSimulationPipeline` in RendererVK. Bakes the fine SHORE MAP (height/water/flow; overrides the terrain-data map inside its range) for shoaling, surf, swash and the VS land cull. `sampleWaterHeight` = CPU buoyancy field from the GPU displacement readback (App wires it into `PhysicsWorld::setWaterSurface`). WIND CONVENTION: the spectrum's dominant term travels AGAINST `windDirection` ŌĆö the swell heading is `swellTravelAngle()` = wind angle + pi; `steeredWindAngle` turns the SIM wind toward the baked shore flow around the camera (rate-limited slew, votes from the CPU shore copy) so waves roll inland ŌĆö per-pixel domain rotation is disabled in ocean_wave.inc.glsl (creases; see the comment there before reviving it)
* `ScatterSystem` (Tweaks "Scatter"): trees/rocks/grass over the terrain ŌĆö config is two tables in Scattering.cpp (`SCATTER_ASSETS`: .oc-referenced models with footprint/scale/slope settings; `SCATTER_RULES`: climate ATTRACTORS in real units (C, mm/yr) with Gaussian density falloff ŌĆö no biome enum). Cell placements computed deterministically on a worker from the SAME sampler the terrain renders (footprint dart-throw, cluster noise, slope/altitude bands), bridged to RenderNodes; per-rule `viewDistance` spawns/despawns instance groups without regenerating cells; everything regenerates when the sampler identity changes

## Network
* Winsock UDP/TCP + a reliable-UDP game protocol (depends only on Core; Ws2_32 + Bcrypt linked PRIVATE). Module partitions `:Address`, `:Serialization`, `:Socket`, `:Crypto` (internal, nothing exported), `:Reliable`, re-exported from `Public/Network.ixx`. IPv4 only, wire format is raw little-endian bytes ├óŌé¼ŌĆØ no versioning/compat, both ends must match (`protocolId`)
* `:Serialization`: `NetWriter`/`NetReader` over a caller-provided byte span ├óŌé¼ŌĆØ trivially-copyable `write<T>`/`read<T>`, varint (LEB128 + zigzag), strings, unorm/snorm/`writeQuantized<UInt>(v, min, max)` float quantization; overflow-safe (check `overflowed()`), reads are zero-copy views
* `:Socket`: non-blocking RAII wrappers ├óŌé¼ŌĆØ `UdpSocket` (big SO_RCVBUF, SIO_UDP_CONNRESET disabled, optional broadcast), `TcpSocket`/`TcpListener` (TCP_NODELAY, `poll()` resolves connects), `TcpMessageStream` (uint32-length-framed messages with partial send/recv buffering), `netResolveHost` (blocking DNS). Winsock initialized lazily on first open
* `:Reliable`: `NetHost` ├óŌé¼ŌĆØ symmetric client/server/p2p over one `UdpSocket` (two hosts `connect()`ing to each other resolve the simultaneous handshake, tie-broken on connect salts). Poll model like the rest of the engine: queue with `send(peer, bytes, ENetDelivery, channel)`, per-frame `update(dt)` pumps/retransmits/flushes (messages batched into ├óŌĆ░┬ż`maxPacketSize` packets), drain `takeEvents()` (Connected/Disconnected/Message). Deliveries: Unreliable, UnreliableSequenced (stale-dropped), Reliable (ordered + guaranteed, acked via packet seq + 32-bit ack bitfield piggybacked on every packet, RTT-based retransmit, fragments transparently up to `maxMessageSize`) ├óŌé¼ŌĆØ on up to 8 independent channels (index rides in the message kind byte, zero wire cost) so bulk transfers don't head-of-line-block other reliable streams. 4-way anti-spoof challenge handshake (responder is stateless until the challenge round-trips: `serverSalt = HMAC(hostSecret, addr|salt|flags)`); `NetHostConfig::encrypt` adds ephemeral ECDH P-256 key agreement in the handshake + AES-128-GCM per payload packet (+16B tag, nonce = sender role + implicit 64-bit packet counter; Windows CNG, hardware AES) ├óŌé¼ŌĆØ secure against spoofing/eavesdropping/tampering, not an active MITM (unauthenticated keys). Keepalives/timeouts, per-peer RTT/loss stats, and live-editable link simulation (`simPacketLoss`/`simLatencyMs`/`simJitterMs`) in `NetHostConfig`

## Audio
* miniaudio (output device, mixing, node graph, decoding; `MINIAUDIO_IMPLEMENTATION` in Audio.cpp) + Steam Audio (HRTF binaural spatialization, statically linked `phonon(d).lib`), wrapped in `module Audio`; depends only on Core, neither API leaks into the public surface (handles store opaque pointers)
* `Globals::audio` (`AudioSystem`): `initialize()` opens the default device (48kHz stereo) and creates the Steam Audio context/HRTF; `setListener(pos, forward, up[, velocity])` once per frame from the camera (main.cpp does this next to `scriptContext.update`) ├óŌé¼ŌĆØ it recomputes every source's HRTF direction, inverse-clamped distance attenuation, and doppler on the game thread (audio thread reads them via atomics). Master volume is a Tweak under Audio/System
* Mono sounds route ma_sound ├óŌĆĀŌĆÖ per-source custom `BinauralNode` (fixed 256-frame `iplBinauralEffectApply` blocks behind small FIFOs) ├óŌĆĀŌĆÖ endpoint; stereo or `setRelative(true)` sounds play unspatialized. Steam Audio's scene-based occlusion/reflection features are unused so far ├óŌé¼ŌĆØ only the HRTF direct path
* `AudioBuffer`/`AudioSource`: RAII movable handles, like PhysicsBody. `loadSound(path)` (WAV/FLAC/MP3, relative to `Assets/`) or `createBuffer(EAudioFormat, pcmBytes, sampleRate)` for procedural sounds (everything converts to f32 internally); buffers must outlive sources playing them (destroying one stops those sources). Sources: play/pause/stop, looping/gain/pitch/position/velocity, `setAttenuation`
* `playOneShot(buffer, worldPos, gain, pitch)` / `playOneShot2D` = fire-and-forget from an internal 32-source pool (steals the oldest slot when full). Key 8/9 in the App plays a procedural blip at the thrown body's spawn position as a smoke test
* Gameplay integration lives in Entity, not here: `AudioComponent` (see Entity) + the script nodes "Trigger Audio" (dynamic exec entry pin per sound alias, special-cased in NodeDef.ixx/Scene.cpp; connected Position/Volume/Pitch inputs override the authored settings via `ctx->entityTriggerAudio`'s overrideMask) and "Stop Audio"
* Skeletal animation runtime (depends only on Core). Partitioned module `Animation` (`import Animation;`): `Animation:Skeleton` (`Skeleton` bone hierarchy), `Animation:Clip` (`AnimationClip`/`AnimationSet` keyframes + `AnimationPlayer`: CPU sampler producing a bone-matrix palette, with crossfade blending, 1D `BlendSpace1D`, and programmatic per-bone posing via `setBoneTransform`/`setBoneOffset`), and `Animation:StateMachine` (`AnimStateMachine`: parameter-driven states/transitions that crossfade an `AnimationPlayer`)
* File builds `Skeleton`/`AnimationClip` from Assimp; RendererVK consumes the palette for GPU skinning; Entity's `AnimatorComponent` + .apl assets drive it data-driven (World caches retargeted clip sets per skeleton+animator)

## File
* Wrappers for assets and file system related functionality
* `ISceneData::createAssimpLoader()` for 3d model files (Assimp), `ISceneData::createProceduralLoader()` to programmatically generate assets (terrain, skysphere, debug shapes)
* Cooked scene cache: `ISceneData::loadCached(path, mergeNodes, preTransform, SceneCookOptions)` (World uses it for every non-procedural container) serves a binary snapshot (`Assets/Local/Cooked/<stem>_<hash>.vsc`, `File:CookedSceneData` reader + `SceneCooker.cpp` writer) validated by magic/version, source mtime+size, an options hash (Assimp flags + LOD gen params), and per-converted-texture source stamps; any mismatch re-imports and re-cooks in place. Cooking bakes meshopt LOD index chains (`IMeshData::getNumLodLevels`/`getLodIndices`; ObjectContainer consumes them instead of simplifying at load, so changing the LOD tweaks recooks on next start) and converts non-DDS textures referenced by diffuse/normal/metalRoughness slots (loose png/tga/jpg AND embedded) to BC1/BC3(alpha)/BC5(normals) .dds next to the .vsc so they mip-stream (alpha-MASKED materials' diffuse mips are alpha-rescaled to preserve level-0 alpha-test coverage) (stb_image + stb_image_resize2 + stb_dxt compiled STB_*_STATIC in File; RendererVK owns the extern stb_image). Skinned/animated scenes always bypass the cache (skeleton/clips are not serialized). Delete `Assets/Local/Cooked/` to force a full recook
* `AssetParser` (`File:AssetParser`): the text format behind all engine text assets (.pre/.oc/.anm/.apl). Indentation-defined hierarchy of `AssetNode` (key + values + children), `#`/`//` comments, quoted strings, comma-separated vectors; `loadAssetFile`/`parseAssetText`/`writeAssetText`, case-insensitive `find`, typed accessors with fallbacks. `File:ObjectDescription` parses .oc declarations into `ObjectContainerDesc` (`loadObjectContainerDesc`) ŌĆö lives here (not Entity) so Procedural's scatter assets import through .oc files too
* `FileSystem` sets the working directory to `Assets/` at startup

* Panels:
	* Viewport ├óŌé¼ŌĆØ renderer output; assets can be drag-dropped from Content to spawn at the cursor (`Camera::screenToWorld`)
	* Scene (`SceneView`) ├óŌé¼ŌĆØ live hierarchy: selection, F2 rename, Delete, drag-reparent, drag-to-Content to save a prefab, context menus, prefab instances shown locked (unpackable)
	* Properties (`PropertiesPanel`) ├óŌé¼ŌĆØ edit selected entity's name/enabled/pos/scale/rot, render component info + bounds toggle, script path
	* Content (`AssetBrowser`) ├óŌé¼ŌĆØ filesystem browser rooted at `Assets/`: grid/list, drag assets into viewport/hierarchy, accept prefab drops (overwrite/cycle guards), create/rename .scr files, context menus
	* Script (`NodeEditor::Scene` + imgui-node-editor) ├óŌé¼ŌĆØ the visual scripting frontend: categorized node palette (spawn submenus, link-drag spawning, pin redirect), comment boxes, waypoints, copy/paste, generates C++ (`generateCpp`) and saves .scr (code + `//@graph` metadata), Compile & Run requests drained by main (`takeScriptReloadRequests`), follows the hierarchy selection to open its entity's script (unsaved-changes prompt)
	* Log (`OutputLog`) ├óŌé¼ŌĆØ console view of `Core.Log` with level filters/search
	* Tweaks (`TweakPanel`) ├óŌé¼ŌĆØ auto-populated from the Tweak registry
	* Stats ├óŌé¼ŌĆØ renderer statistics (`ui.setRenderStats`)

# Asset files (all text formats parsed via `AssetParser`, registered by `AssetRegistry`)
* `.oc` (ObjectContainer): a model source + import options (`Path`, `MergeNodes`, `PreTransformVertices`, `DecimationFactor` (< 1 meshopt-simplifies base geometry at cook time; cache-only, so skinned/procedural containers never decimate), `MaterialOverrides` with `PipelineIdx`/`ExcludeFromRayTracing`/texture overrides) plus named `StaticMesh`/`SkinnedMesh` spawnable entries (`Node` path within the model; SkinnedMesh adds `Type Humanoid/Generic` + optional bone mapping). Skinned containers MUST keep `PreTransformVertices false`
* `.pre` (Prefab): a named entity (hierarchy) ├óŌé¼ŌĆØ `Prefab <name>` root with `Component Render/Animator/Script/Physics/Audio/Scene` children; `Component Scene` nests child `Prefab` references. `Component Audio` nests `Sound <alias>` entries, each with an optional `Select` and one or more `Path <file>` clips whose own child lines are that clip's playback settings. Render references a spawnable (`StaticMesh <name>`) or `ObjectContainer <name>` + `Node`, plus `Position/Rotation/Scale`. Spawn by prefab name (`world.spawn("physicsCube", ...)`) or file path (`world.spawnAssetFile`)
* `.anm`: `Animation <name>` clip entries ├óŌé¼ŌĆØ source `ObjectContainer` (fbx), `Loop`, `Skip` (channels), `Event <name> <normalizedTime>` notifies (fired into the entity's script/animator `onEvent`)
* `.apl`: `Animator <name>` graph ├óŌé¼ŌĆØ `Parameter`s (Float/Bool/Trigger), `Clip` bindings, `BlendSpace1D` with `Sample`s, `StateMachine` (states with `Play`/`SpeedParam`/`SpeedScale`, `Transition`/`AnyTransition` with `Condition`/`ExitTime`/`Fade`)
* `.scr`: visual script = generated C++ + `//@graph` node metadata (see Script)
* `Assets/Local/` is generated output (compiled SPIR-V + shader source dumps for Aftermath, compiled script DLLs/PDBs, cooked scene caches `Cooked/*.vsc` + converted `.dds`) ├óŌé¼ŌĆØ never hand-edit
